MeshAPI Documentation

OpenAI-compatible API endpoint providing free access to 25+ models including Claude, GPT-4o, and Gemini. Drop-in replacement for any OpenAI SDK or tool.

Introduction

MeshAPI is an OpenAI-compatible proxy that routes requests to multiple LLM providers. It uses the standard POST /v1/chat/completions format, so any tool or library that supports OpenAI works with zero changes.

All requests are proxied through the MeshAPI demo endpoint. You are not billed by model providers directly. Usage is subject to availability of the upstream demo service.

Base URL

https://your-deployment.vercel.app/v1

Or for local development: http://localhost:4321/v1

Authentication

All API requests require a valid API key passed in the Authorization header as a Bearer token:

Authorization: Bearer sk-mesh-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

API keys are managed through the admin dashboard. Keys have the format sk-mesh- followed by 40 hex characters.

Admin Dashboard

Navigate to /login to access the admin dashboard where you can create keys, set per-key limits, monitor usage, and revoke access.

Quick Start

curl -X POST "http://localhost:4321/v1/chat/completions" \
  -H "Authorization: Bearer sk-mesh-your-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "anthropic/claude-sonnet-5",
    "messages": [{"role": "user", "content": "Say hello!"}],
    "max_tokens": 100
  }'
100% compatible with the OpenAI Python SDK, Node.js SDK, and any tool that supports custom base URLs. Just change the base URL and API key — no other code changes needed.

POST /v1/chat/completions

The main chat endpoint. Fully compatible with the OpenAI Chat Completions API format.

Request Body

ParameterTypeRequiredDescription
modelstringRequiredModel ID from the /v1/models list
messagesarrayRequiredArray of {role, content} objects (system/user/assistant)
max_tokensintegerOptionalMaximum tokens to generate (default: 2048)
temperaturenumberOptionalSampling temperature 0–2 (default: 1.0)
streambooleanOptionalStream response via SSE (default: false)
top_pnumberOptionalNucleus sampling parameter
stopstring/arrayOptionalStop sequences

Non-streaming Response

{
  "id": "chatcmpl-abc123",
  "object": "chat.completion",
  "created": 1700000000,
  "model": "anthropic/claude-sonnet-5",
  "choices": [{
    "index": 0,
    "message": { "role": "assistant", "content": "Hello! How can I help you today?" },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 12,
    "completion_tokens": 9,
    "total_tokens": 21
  }
}

Python

Use the official openai Python SDK with a custom base_url:

from openai import OpenAI

client = OpenAI(
    api_key="sk-mesh-your-key-here",
    base_url="http://localhost:4321/v1"
)

response = client.chat.completions.create(
    model="anthropic/claude-sonnet-5",
    messages=[{"role": "user", "content": "Hello!"}],
    max_tokens=512
)

print(response.choices[0].message.content)

Async / Streaming

async with client.chat.completions.stream(
    model="openai/gpt-4o",
    messages=[{"role": "user", "content": "Tell me a story"}]
) as stream:
    async for chunk in stream:
        print(chunk.choices[0].delta.content or "", end="")

OpenCode

Add this to your OpenCode config (~/.config/opencode/config.json):

{
  "providers": {
    "meshapi": {
      "type": "openai",
      "apiKey": "sk-mesh-your-key-here",
      "baseURL": "http://localhost:4321/v1",
      "models": {
        "anthropic/claude-sonnet-5": { "name": "Claude Sonnet 5 (Free)" },
        "openai/gpt-4o": { "name": "GPT-4o (Free)" }
      }
    }
  }
}

Cursor / Continue.dev

Cursor: Go to Settings → Models → Add Model. Set the model name to your preferred model ID, and under OpenAI API Key enter your API key. Then toggle Override OpenAI Base URL and enter http://localhost:4321/v1.

Continue.dev: Add to ~/.continue/config.json:

{
  "models": [{
    "provider": "openai",
    "model": "anthropic/claude-sonnet-5",
    "title": "Claude (Free via MeshAPI)",
    "apiKey": "sk-mesh-your-key-here",
    "apiBase": "http://localhost:4321/v1"
  }]
}

Aider

export OPENAI_API_KEY="sk-mesh-your-key"
export OPENAI_API_BASE="http://localhost:4321/v1"

aider --model "openai/anthropic/claude-sonnet-5" --openai-api-key $OPENAI_API_KEY

Or add to ~/.aider.conf.yml:

openai-api-key: sk-mesh-your-key
openai-api-base: http://localhost:4321/v1
model: openai/anthropic/claude-sonnet-5

Errors

StatusCodeCause
401unauthorizedMissing or invalid Authorization header
403revokedAPI key has been revoked
429rate_limitedDaily request or token limit reached
400bad_requestMissing model or messages in body
502upstream_errorMeshAPI upstream returned an error
500internal_errorUnexpected server error

Error Response Format

{
  "error": {
    "message": "API key has been revoked",
    "type": "invalid_api_key",
    "code": "revoked"
  }
}

Rate Limits

Limits are set per-key in the admin dashboard. Three types of limits are supported:

Limit TypeDescriptionReset
requests_per_dayMaximum API calls per calendar dayDaily at 00:00 UTC
tokens_per_dayMaximum total tokens (prompt + completion) per dayDaily at 00:00 UTC
tokens_totalLifetime token cap — key stops working when reachedNever (manual reset required)

If no limit is set for a type, it is treated as unlimited. When a limit is reached the API returns HTTP 429.
